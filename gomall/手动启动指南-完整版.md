# Gomall 服务手动启动指南

## 环境信息

- **本地开发**: 所有服务运行在本地
- **MySQL 密码**: 123456
- **适用平台**: macOS (Apple Silicon) / Windows / Linux

---

## 第一步：检查基础设施服务

### 1.1 检查 Docker 是否运行

```bash
# macOS / Linux
docker info

# Windows (PowerShell)
docker info
```

### 1.2 检查 Docker 容器状态

```bash
docker ps
```

应该看到：MySQL, Redis, Consul, Jaeger, Prometheus, Grafana 等容器运行中。

### 1.3 如果容器没有运行

```bash
docker-compose up -d
```

### 1.4 测试 MySQL 连接

```bash
# macOS / Linux
docker exec gomall-mysql-1 mysql -uroot -p123456 -e "SELECT 1"

# Windows (PowerShell)
docker exec gomall-mysql-1 mysql -uroot -p123456 -e "SELECT 1"
```

看到输出 `1` 说明 MySQL 正常。

---

## 第二步：启动应用服务

**重要提示**：

- 按顺序启动服务（有依赖关系）
- 每个服务启动后等待 30-60 秒
- 每个服务在独立的终端窗口运行
- 内存不足时可以只启动需要的服务

### 启动顺序

1. user → 2. product → 3. cart → 4. payment → 5. checkout → 6. order → 7. email → 8. frontend

---

## 启动命令

### macOS / Linux

```bash
# 终端 1 - User 服务 (端口 8881)
cd app/user && go run .

# 终端 2 - Product 服务 (端口 8882)
cd app/product && go run .

# 终端 3 - Cart 服务 (端口 8883)
cd app/cart && go run .

# 终端 4 - Payment 服务 (端口 8884)
cd app/payment && go run .

# 终端 5 - Checkout 服务 (端口 8885)
cd app/checkout && go run .

# 终端 6 - Order 服务 (端口 8886)
cd app/order && go run .

# 终端 7 - Email 服务 (端口 8887)
cd app/email && go run .

# 终端 8 - Frontend 服务 (端口 8080)
cd app/frontend && go run .
```

### Windows (PowerShell)

```powershell
# 窗口 1 - User 服务
cd app\user
go run .

# 窗口 2 - Product 服务
cd app\product
go run .

# ... 以此类推
```

等待看到类似日志：
- Kitex 服务: `[Info] KITEX: server listen at addr=[::]:8881`
- Hertz 服务: `[Info] HERTZ: server listen at addr=[::]:8080`

## 常见问题

### Q1: MySQL 连接失败 (Error 1045)

**错误信息**：`Access denied for user 'root'@'127.0.0.1'`

**解决方法**：

1. 检查 .env 文件中的密码是否为 `123456`
2. 重启 MySQL 容器：

```bash
docker-compose restart mysql
```

### Q2: 服务启动后立即退出

**可能原因**：配置文件错误或端口被占用

**解决方法**：

1. 查看服务窗口的错误信息
2. 检查端口是否被占用：

```bash
# macOS / Linux
lsof -i :8881

# Windows
netstat -ano | findstr "8881"
```

### Q3: Frontend 返回 502 错误

**原因**：后端 RPC 服务未完全启动

**解决方法**：

1. 确保所有 RPC 服务（user, product, cart 等）都已启动
2. 等待 1-2 分钟让服务完全初始化
3. 检查 Consul UI 查看服务注册情况：http://localhost:8500

### Q4: 内存不足

**解决方法**：

1. 不要同时启动所有服务
2. 只启动必要的服务：user, product, frontend
3. 关闭其他占用内存的程序

## 停止服务

### 停止单个服务

在对应的终端窗口按 `Ctrl+C`

### 停止所有服务

```bash
# macOS / Linux
chmod +x stop-services.sh
./stop-services.sh

# 或手动停止
docker-compose down
```

## 最小启动配置（内存不足时）

如果内存不足，可以只启动核心服务：

1. **User 服务** (8881) - 必需
2. **Product 服务** (8882) - 必需
3. **Frontend 服务** (8080) - 必需

这样可以访问基本的商品浏览功能。

如果需要购物车和结账功能，再依次启动：
4. **Cart 服务** (8883)
5. **Checkout 服务** (8885)
6. **Payment 服务** (8884)
7. **Order 服务** (8886)
